# Functional requirement blocks

Add cross-cutting capability requirements here when they apply to multiple
systems (authentication, user journeys, orchestration, etc.).

- Canonical baseline: `docs/templates/3layer/functional_requirements.md`
- Localised work: duplicate `_templates/Local_SRS_Template.md` to capture team-specific deltas.

When you introduce a new block, document:

1. The persona/role that owns it.
2. Acceptance criteria or KPIs.
3. Traceability to related NFR or integration requirements.
